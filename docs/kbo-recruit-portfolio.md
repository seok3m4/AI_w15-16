## 1. 15초 소개

저는 야구팬이 실제로 앱에서 정보를 찾는 흐름을 보고, 그 흐름을 화면과 API로 직접 구현해보는 개발자 지망생입니다.

이번 프로젝트에서는 KBO 팬이 오늘 경기 하나를 볼 때 필요한 정보가 여러 메뉴에 흩어져 있다는 점에 집중했습니다. 경기 일정, 선발투수, 라인업, 문자중계, 박스스코어, 뉴스, 팬 의견을 `경기방` 중심으로 묶고, 웹 게시판과 모바일 앱형 화면, Android APK까지 만들어봤습니다.

## 3. 프로젝트 한눈에 보기

### 프로젝트명

KBO Talk / KBO Fan Hub

### 한 줄 설명

KBO 경기 정보와 팬 커뮤니티를 한 흐름으로 묶은 야구 게시판 겸 모바일 앱형 서비스입니다.

### 만들게 된 이유

야구팬은 보통 "오늘 어느 팀 경기하지?"에서 출발합니다. 경기 전에는 선발투수와 라인업을 보고, 경기 중에는 문자중계를 확인하고, 경기 후에는 박스스코어와 뉴스, 팬 반응을 봅니다.

그런데 이 정보들이 서로 떨어져 있으면 팬은 계속 화면을 오가야 합니다. 저는 이 문제를 `경기방`이라는 단위로 묶어보고 싶었습니다.

### 내가 맡은 역할

개인 프로젝트로 진행했기 때문에 기획, 화면, API, DB, 배포, Android WebView 연결까지 모두 직접 했습니다.

- 서비스 흐름 설계
- 게시판 화면과 모바일 앱형 화면 구현
- Next.js API 구현
- PostgreSQL / Prisma 데이터 모델링
- RAG, MCP, Agent 기반 글쓰기 보조 기능 구현
- Supabase / Vercel 배포
- Android WebView APK 빌드와 GitHub Release 업로드

## 6. 트러블슈팅

### 6.1 PostgreSQL pgvector 환경 문제

문제:

RAG 유사 글 검색을 위해 게시글 임베딩을 저장하려고 했지만, 로컬 PostgreSQL에 `pgvector` 확장이 설치되어 있지 않아 migration이 실패했습니다.

시도:

- 일반 PostgreSQL 테이블에 벡터를 문자열로 저장하는 방법을 검토했습니다.
- 별도 Vector DB를 붙이는 방법도 생각했습니다.
- 하지만 프로젝트 규모상 DB를 하나로 유지하는 편이 더 단순하다고 판단했습니다.

해결:

PostgreSQL에 pgvector 확장을 설치하고, Supabase 배포 DB에서도 vector 확장을 활성화했습니다. 이후 게시글 작성/수정 시 임베딩을 생성하고 pgvector에 저장하는 흐름으로 정리했습니다.

배운 점:

AI 기능은 모델 호출만으로 끝나지 않고, DB 확장과 배포 환경까지 같이 맞아야 동작한다는 것을 알게 됐습니다.

### 6.2 Vercel 배포 후 로그인 상태 문제

문제:

로컬에서는 로그인 후 사용자 상태가 잘 반영됐지만, 배포 환경에서는 쿠키와 인증 상태 반영이 기대와 다르게 보이는 상황이 있었습니다.

시도:

- `/api/auth/me` 응답을 확인했습니다.
- HttpOnly Cookie 설정을 확인했습니다.
- 배포 환경에서 새로고침과 페이지 이동 시 상태가 어떻게 바뀌는지 확인했습니다.

해결:

로그인 후 사용자 정보를 다시 요청하고, 인증 상태 확인 중인 화면을 분리해 배포 환경에서도 상태가 안정적으로 반영되도록 개선했습니다.

배운 점:

인증은 API 하나가 성공하는 것만으로 끝나지 않고, 브라우저 쿠키, 서버 응답, 클라이언트 상태가 같이 맞아야 한다는 점을 경험했습니다.

### 6.3 KBO 데이터가 항상 같은 형태로 오지 않는 문제

문제:

KBO 공식 API 문서가 공개된 형태가 아니라, 경기 정보와 라인업, 기록 데이터를 가져올 때 항상 같은 수준의 데이터가 보장되지 않았습니다.

시도:

- 라인업이 없는 경기와 있는 경기를 나눠 확인했습니다.
- 종료 경기와 예정 경기를 나눠 확인했습니다.
- 데이터가 없는 경우 무조건 빈 화면을 보여주는 대신 상태 메시지를 보여주도록 바꿨습니다.

해결:

경기 상태에 따라 선발투수, 라인업, 승리투수/패전투수/세이브 투수, 승부 예측 버튼 노출 여부를 다르게 처리했습니다.

배운 점:

외부 데이터 연동은 "데이터를 가져오는 코드"보다 "데이터가 없거나 늦게 나오는 상황을 어떻게 보여줄지"가 더 중요하다는 것을 느꼈습니다.

## 7. 기술 스택과 선택 이유

숙련도 바 대신 실제 사용 경험과 선택 이유를 중심으로 정리했습니다.

| 기술 | 사용한 곳 | 선택 이유 / 경험 |
| --- | --- | --- |
| Next.js | 웹 화면, API Routes, 배포 | 화면과 서버 API를 한 프로젝트에서 관리할 수 있어 개인 프로젝트에 적합했습니다. |
| React | 게시판 UI, 경기방 UI, 모바일 앱형 화면 | 상태 변화가 많은 화면을 컴포넌트 단위로 나눠 구현했습니다. |
| TypeScript | API 응답, 컴포넌트 props, DB 타입 | 게시글, 댓글, 경기 데이터처럼 구조가 있는 데이터를 안정적으로 다루기 위해 사용했습니다. |
| PostgreSQL | 사용자, 게시글, 댓글, 태그, 추천, 기록 저장 | 관계형 데이터와 검색 조건이 많은 게시판에 적합했습니다. |
| Prisma | DB 모델링, migration, query | DB schema와 TypeScript 타입을 연결해 실수를 줄였습니다. |
| pgvector | 유사 게시글 검색 | 게시글 임베딩을 DB 안에서 검색하기 위해 사용했습니다. |
| OpenAI API / LangChain.js | 임베딩, 요약, 리뷰 초안 | 직접 만든 게시글 데이터를 바탕으로 글쓰기 보조 기능을 만들었습니다. |
| Node.js JSON-RPC 구조 | 뉴스 URL, 경기 정보 도구 호출 | 외부 데이터를 도구처럼 분리해 호출하는 구조를 연습했습니다. |
| Vercel / Supabase | 웹 배포, PostgreSQL 운영 DB | 배포 URL과 운영 DB를 실제로 연결해 데모 가능한 상태로 만들었습니다. |
| Android WebView | 모바일 앱형 APK | 웹 서비스를 앱 형태로 보여주고 APK 배포까지 확인했습니다. |

## 8. 직무와 연결되는 부분

### KBO 앱 서비스 기획 직무와의 연결

- 앱 메뉴와 서비스 구조를 팬 사용 흐름 기준으로 다시 정리했습니다.
- 경기, 기록, 뉴스, 커뮤니티 참여를 경기방 중심으로 연결했습니다.
- 화면만 그린 것이 아니라 실제 API와 DB로 동작하는 MVP를 만들었습니다.
- 개선 아이디어를 기능 요구사항, 예외 처리, 성공 지표까지 나눠 정리했습니다.

### KBO 앱 데이터 관리, 분석 지원 직무와의 연결

- 게시글, 태그, 조회수, 추천, 댓글 같은 커뮤니티 데이터를 저장하는 구조를 만들었습니다.
- 경기방 진입률, 기록실 조회 수, 뉴스 후 글쓰기 전환율처럼 운영 시 확인할 지표를 정의했습니다.
- Supabase PostgreSQL 기반으로 운영 DB 연결을 경험했습니다.
- AI 개인화 사례를 응원팀 설정, 관련 글 요약, 야구 게시판 도우미 기능으로 작게 구현했습니다.

## 10. 아직 부족한 점

- 실제 KBO 앱 사용자 데이터가 없어서 개선 효과는 가설과 MVP 검증 수준입니다.
- KBO 공식 데이터가 문서화된 API로 제공되는 구조가 아니라 웹 구조 변화에 영향을 받을 수 있습니다.
- Android 앱은 네이티브 전체 구현이 아니라 WebView 기반 MVP입니다.
- 승부 예측은 통계 모델이 아니라 선발투수, 라인업, 순위 정보를 바탕으로 한 참고용 브리핑입니다.

## 12. 면접에서 이야기하고 싶은 점

1. 왜 단순 게시판이 아니라 경기방 중심 구조로 바꿨는지
2. KBO 데이터가 불완전할 때 화면에서 어떻게 처리했는지
3. RAG, MCP, Agent를 기술 과시가 아니라 사용자 흐름 안에 숨기려고 한 이유
4. WebView 기반 APK를 만들면서 웹과 앱의 경계를 어떻게 이해했는지
5. 실제 운영한다면 어떤 지표를 보고 개선할지

## 13. 시연 순서

1. 웹 메인에서 게시판, 오늘의 경기, 인기글 확인
2. 경기방에서 선발투수, 라인업, 문자중계, 박스스코어 확인
3. 기록실에서 팀 순위와 선수 기록 확인
4. 뉴스 페이지에서 KBO 뉴스와 URL 정리 흐름 확인
5. 글쓰기에서 태그, 이미지 첨부, 비슷한 글 보기, 리뷰 초안 작성 확인
6. 댓글 작성 시 부적절한 표현 경고 확인
7. 모바일 앱형 화면과 Android APK 배포 설명
